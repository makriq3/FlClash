# Журнал изменений

Этот файл используется как источник release notes для GitHub Releases. Для нового релиза добавляйте секцию `## vX.Y.Z` в начало файла, а первые 3-5 bullets формулируйте как короткие пользовательские highlights.

## v0.9.00

### Обновления и канал релизов

- Репозиторий, ссылки приложения и release-метаданные окончательно переведены на `makriq-org/FlClash` и `makriq`.
- Проверка обновлений получила отдельный pre-релизный переключатель в настройках; по умолчанию он выключен, а stable-канал остаётся безопасным.
- Ручная и автоматическая проверка обновлений теперь используют единый выбор релизного канала и корректно различают stable и prerelease.

### Надёжность релизов

- Сравнение версий обновлено так, чтобы корректно обрабатывать теги вида `0.9.00-pre1` и переходы с prerelease на stable.
- Android updater выбирает совместимый ABI-артефакт и использует `sha256` для проверки скачанного файла.
- Release workflow публикует артефакты и контрольные суммы согласованно для поддерживаемых платформ, а Linux RPM остаётся только для stable-релизов.

## v0.9.00-pre1

### Перенос репозитория и контактов

- Публичные ссылки, badges, issue templates и release helper-скрипты переведены на новый адрес `makriq-org/FlClash`.
- Упоминания владельца и packaging-контакты очищены от старого аккаунта `makriq3` и приведены к `makriq`.
- Встроенный repository id приложения синхронизирован с новым репозиторием, чтобы About и release-ссылки больше не вели в старый namespace.

### Предрелизный контур

- Сравнение версий обновлено так, чтобы корректно обрабатывать prerelease-форматы и теги вида `0.9.00-pre1`.
- В настройках добавлен отдельный переключатель для pre-релизного канала обновлений; по умолчанию он выключен.
- Release workflow теперь публикует теги `v*-pre*` как GitHub prerelease с поддерживаемым набором артефактов и `*.sha256`, оставляя Linux RPM только для stable-релизов.
- Документация релизного процесса обновлена под новый репозиторный контекст и отдельный prerelease-поток.

## v0.8.94

### Доработка Android-маршрутизации и защиты

- Исправлена регрессия Android-маршрутизации, появившаяся после первой волны защиты localhost.
- Восстановлена корректная прямая маршрутизация для доменных Android-правил на усиленном TUN-пути без повторного открытия локального прокси на localhost.
- Восстановление доменной маршрутизации Android VPN включено на усиленном TUN-пути через безопасный `sniffer` для `http`, `tls` и `quic`.
- Совместимые значения Android `bypassDomain` преобразованы в приоритетные правила Clash `DIRECT` во время генерации усиленного Android VPN-профиля.

### Согласованность runtime-защиты

- Защита Android localhost listeners теперь удерживается не только на этапе настройки профиля, но и во время обновления `updateConfig(...)` на лету.
- Синхронизация runtime-конфига больше не может повторно открыть `mixed-port`, `port`, `socks-port`, `redir-port`, `tproxy-port`, `allow-lan` или `external-controller`, пока активна защита Android.
- Сохранена согласованная семантика Android TUN-маршрутизации, включая обработку `route-address` и `RouteMode.bypassPrivate`, через единый runtime-путь защиты.

### Регрессионное покрытие и безопасность релизов

- Добавлено прицельное регрессионное покрытие для конфликта маршрутизации и защиты, проявившегося после `v0.8.93`.
- Добавлены тесты для:
  - закрытия локальных listeners в усиленном Android runtime-конфиге,
  - Android TUN-маршрутизации при `RouteMode.bypassPrivate`,
  - платформенно-стабильного разрешения TUN в CI и Android runtime,
  - сохранения явных настроек `sniffer` при одновременном принудительном восстановлении доменной маршрутизации.
- Документация репозитория обновлена так, чтобы она описывала финальную модель защиты Android: runtime-защиту, восстановленную корректность маршрутизации и анти-регрессионные предохранители.

## v0.8.93

### Защита Android VPN

- Добавлен Android-only слой защиты, который переписывает сгенерированную Clash runtime-конфигурацию в VPN-режиме.
- Все локальные inbound listener-порты в Android VPN-режиме теперь принудительно закрываются:
  - `port=0`
  - `socks-port=0`
  - `mixed-port=0`
  - `redir-port=0`
  - `tproxy-port=0`
- В усиленном Android VPN-режиме `external-controller=''` теперь выставляется принудительно, а `allow-lan=false` удерживается постоянно.
- Собственные Android-запросы приложения больше не завязаны на localhost mixed proxy при активном VPN.

### Снижение распознаваемости

- IPv4 и IPv6 адреса Android-туннеля теперь рандомизируются при каждом запуске VPN.
- Статичный `VpnService` session label заменён на нейтральный `VPN`.

### Верификация и автоматизация релизов

- Добавлены регрессионные тесты для защиты Android VPN.
- Добавлен отдельный веточный Android workflow сборки для GitHub Actions.
- Основной workflow релиза обновлён так, чтобы этот форк мог собираться и публиковаться из собственного репозиторного контекста.
- Настройка подписи Android сделана условной по секретам репозитория, чтобы сопровождение форка было проще.

### Независимость репозитория

- Основной README переписан вокруг целей этого форка по усилению приватности.
- В репозиторий добавлены исследование защиты Android VPN и документация по безопасности.
- Метаданные релизов и шаблоны обновлены так, чтобы форк больше не зависел от апстримного брендинга и инфраструктурных предположений.
